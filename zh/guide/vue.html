# 前端

# 用qiankun来实现vue技术栈的前端微服务

qiankun的开发和打包和正常模式一模一样，它使用一个全局变量__POWERED_BY_QIANKUN__来区分微前端模式和正常模式，不需要额外的配置和代码。

现有的模块有：

  • 基座（main）
  • 组织模型管理+基础功能设置（system）
  • 在线开发（form）
  • BPM协同（workflow）
  • 消息管理（message）
  • 系统监控（monitoring）
  • 数据可视化（report）

基座目前包括：

  • 登录页
  • 401、404页面
  • 公用组件 (将项目遇到的公用组件提取出来放在基座，避免每个模块重复加载)
  • 生产环境公用依赖 (public/cdn， 项目生成环境使用externals，减少重复依赖)
  • 主页
  • 个人中心
  • 业务资源配置页面
  • 流程配置页面
  • 机构、用户、角色、岗位等详情页

# 运行和打包

在根目录下，先安装依赖： npm install，再执行npm run install-all为所有项目安装依赖，最后执行npm run dev-all即可启动所有的项目。

npm run build-all可以打包所有vue项目,为方便部署,打包目录结构如下：

└── html/ # 根文件夹 | ├── child/ # 存放所有微应用的文件夹 | ├── sytem/ # 存放微应用 system 的文件夹 | ├── form/ # 存放微应用 form 的文件夹 | ├── workflow/ # 存放微应用 workflow 的文件夹 | ├── message/ # 存放微应用 message 的文件夹 | ├── monitor/ # 存放微应用 monitoring 的文件夹 ├── index.html # 主应用的index.html ├── css/ # 主应用的css文件夹 ├── js/ # 主应用的js文件夹

主子应用打包配置： 主应用vue.config.js中outputDir设置为：../html 子应用vue.config.js中outputDir设置为../html/child/***

# 微前端的使用

首先在基座项目中安装 qiankun

npm i qiankun -S

安装完毕之后，我们开始来进行配置

# 基座配置（主应用）

微前端的配置在 src/micro 下边

  • apps.js 每个模块的引入

    # apps列表⾥的每个对象拥有五个参数，分别是：

    • name：⼦应⽤的名称，子应用之间必须确保唯一
    • entry：⼦应⽤的⼊⼝地址。通过process.env.NODE_ENV判断当前是开发环境还是生产环境，开发环境entry为子应用本地访问地址（端口号与子应用保持一致），生产环境为子应用部署相对路径
    • container：主应⽤容器
    • activeRule：⼦应⽤的激活规则
    • props：主应⽤需要传递给⼦应⽤的数据，目前将主应用store、Vue、Vuex、等传递过去，后续根据需求添加（子应用使用主应用Vue，避免出现多次加载情况，全局方法共用组件等直接挂载在主应用Vue即可）
  • index.js 微前端的启动和生命周期 在 permission.js 中 调用 start(apps) 去启动微前端。 目前是全部启动，项目打开登录之后，再访问第一个子应用时，会加载所有子应用。
  • component.js 公用组件的引入 因为目前子应用使用的主应用的Vue，所以共用组件直接挂载在Vue上即可，子应用无需引用，直接使用

# 子应用配置

  1. 首先要在src目录下创建一个public-path.js
// public-path.js
if (window.__POWERED_BY_QIANKUN__) {
  // 动态设置 webpack publicPath，防止资源加载出错
  __webpack_public_path__ = window.__INJECTED_PUBLIC_PATH_BY_QIANKUN__
}

public-path.js 的目的是为了动态设置webpack的publicPath，防止资源加载出错

  1. 然后在main.js 中引入 public-path.js
let Vue = null
let instance = null
let router = null
const isQiankun = window.__POWERED_BY_QIANKUN__

function render(props = {}) {
  const { container, app, Router } = props
  // 从主应用获取路由进行处理
  const { asyncRoutes } = app
  const routers = asyncRoutes.filter(x => x.name === 'System')
  const routes = setRoutes(routers)

  const dom = container ? container.querySelector('#app') : document.getElementById('app')
  router = new Router({
    // 运行在主应用中时，基础路由地址配置为 /system（和主应用apps里的activeRule保持一致）
    base: isQiankun ? '/system' : '/',
    mode: 'history',
    routes
  })

  instance = new Vue({
    router,
    store,
    render: (h) => h(App)
  }).$mount(dom)
}

if (!window.__POWERED_BY_QIANKUN__) {
  render()
}

// 不存在主应用时可直接单独运行
export async function bootstrap() {
  console.log('system---bootstraped阶段')
}

// 应用每次进入都会调用 mount 方法，通常我们在这里触发应用的渲染方法
export async function mount(props) {
  // 获取主应用传递参数
  const { $parentStore } = props
  Vue = props.Vue
  window.$parentStore = $parentStore

  render(props)
}

//  应用每次 切出/卸载 会调用的方法，通常在这里我们会卸载微应用的应用实例
export async function unmount() {
  console.log('system---unmount阶段')

  instance.$destroy()
  instance.$el.innerHTML = ''
  instance = null
  router = null
}

main.js中bootstrap, mount, unmount 是qiankun框架要求导出的生命周期函数。render是渲染实例函数。

  • bootstrap 主要是数据的初始化
  • mount 是加载完成。这里可以获取到props传过来的参数
  • unmount 模块切出的卸载函数。这里可以去做实例清除等操作。

# 新增子应用

  • 主应用src/micro/apps.js apps 添加子应用注册信息，其中name需确保子应用中唯一，entry的相对路径或端口号唯一，activeRule与访问路径对应

  • 子应用main.js改造，路由的base对应主应用apps设置的activeRule，vue.config.js中publicPath和apps中配置的entry保持一致

  • 子应用src/router/route.js处理子应用路由，在router-map.js里配置路由页面，业务子应用公共组件、公共方法在main中引入，各子应用可根据不同需求安装相应依赖，引入相应方法。

# 代码规范

  • 项目文件命名( js、css、scss、html )： 全部采用小写单词组成， 以下划线分隔
  • 目录结构命名： 建议采用小写单词组成， 以下划线分隔
  • eslint 代码检查

# 交互规范

  1. 提交、暂存等与后台交互的按钮，在点击后禁⽤加loading，等待接⼝响应完成解除禁⽤
  2. 所有输⼊框都需要⻓度控制，与后台⼀致，统⼀按字规划
  3. 所有表单新增与编辑⻚⾯都须有必填校验字段，与后台约束
  4. ⻓度校验：错误提示统⼀使⽤：请输⼊n-m位字符
  5. 必填校验统⼀为：请输⼊/请选择+表单项名称
  6. 选择框和树选择超过五项要做筛选操作(可输⼊进⾏检索)

# 第二步：

根据第一步自动渲染页面并回填值达到查看详情效果

跳转页面路径：'/infoDetail' 文件夹：main/src/views/info-detail/index.vue

获取参数调取接口铺设页面

export default {
  data() {
    return {
      formId: '',
      expand: '',
      formProp: {},
      formList: [],
      resId: '',
      dataId: ''
    }
  },
  mounted() {
    this.resId= JSON.parse(this.$route.query.resId)
    this.dataId= JSON.parse(this.$route.query.dataId)
    this.getFormDetails(this.resId, this.dataId)
  },
  methods: {
    async getFormDetails(val1, val2) {
      const res = await getResourceDetails(val1, val2, {
        appId: this.appId,
        isDetail: true
      })
      const { formInfoVO, formAttrAddList, expand } = res.data
      this.formProp = formInfoVO
      this.expand = expand || ''
      this.formList = formAttrAddList
      this.formId = formInfoVO.formId
    }
  }
}
